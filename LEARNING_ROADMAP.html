0 Foundations — JavaScript & Tooling

~2 weeks

Why Start Here?

Every file in this project is JavaScript. You cannot debug a single line without understanding these basics. Do not skip this phase.

0.1 Modern JavaScript (ES6+)

Beginner 5 days
  • const / let — block-scoped variable declarations (no more var)
  • Arrow Functions — (a, b) => a + b — concise syntax used in every callback
  • Template Literals — `Hello ${name}` — string interpolation used in logging
  • Destructuring — const { url, payload } = job.data — object/array unpacking
  • Spread Operator — { ...obj, newKey: val } — merging/cloning objects
  • Optional Chaining — error.response?.status — safe property access
  • Nullish Coalescing — data.value ?? null — default only for null/undefined
  • Ternary Operator — isFinal ? 'DEAD' : 'FAILED' — inline conditionals
  • Set — new Set([...]) — O(1) membership checks (used in PII redaction)

0.2 Asynchronous JavaScript

Beginner 3 days
  • Callbacks → Promises → async/await — the entire evolution
  • try/catch/finally — error handling with async functions
  • Promise chaining — .then().catch() — used in MongoDB connect
  • AbortController — cancelling async operations (Axios timeout)
  • Event Loop — understand why console.log blocks & Pino doesn't

0.3 Node.js Module System

Beginner 2 days
  • CommonJS — require() / module.exports — used in every project file
  • ES Modules — import/export — used in load test (k6)
  • require.cache — how the tests inject mock Redis
  • package.json — scripts, dependencies, devDependencies, "type": "commonjs"
  • npm install vs npm ci — why Dockerfile uses ci for reproducibility

✅ Phase 0 Checkpoint

You can read any .js file in this project and understand the syntax. You know what async/await, destructuring, and require() do.

5 Security Hardening

~2.5 weeks

⚠️ Why Security Matters Here

This project accepts untrusted URLs from API consumers and makes HTTP requests to them. Without hardening, an attacker could probe your internal network (SSRF), steal credentials, or crash your infrastructure.

5.1 Authentication & Authorization

Intermediate 2 days
  • API Key Auth — x-api-key header checked via safeCompare()
  • Separate Dashboard Token — read-only, never exposes backend key
  • Constant-Time Comparison — crypto.timingSafeEqual defeats timing attacks
  • URL Token Scrubbing — history.replaceState removes ?token= from URL

5.2 HMAC Webhook Signing

Intermediate 2 days
  • What — crypto.createHmac('sha256', secret).update(body).digest('hex')
  • Why — receivers verify the request really came from us
  • Raw Body — sign exact bytes, not re-serialized version
  • transformRequest: [d => d] — prevent Axios from re-serializing

5.3 SSRF Protection

Advanced 3 days
  • DNS-Resolution Based — resolve hostname → check resolved IP vs private ranges
  • Why Not Regex? — 127.0.0.1.nip.io, 0177.0.0.1, 2130706433 all bypass string checks
  • RFC Ranges Blocked — RFC-1918, RFC-5735, RFC-4193, loopback, link-local, CGN
  • DNS Pinning — custom Agent.lookup() prevents TOCTOU / DNS rebinding
  • Fail-Closed — unrecognised IP format → blocked by default

src/worker.js — isPrivateIP() and assertNotPrivate()

5.4 Rate Limiting

Intermediate 1 day
  • express-rate-limit — per-IP sliding window
  • rate-limit-redis — shared counters across replicas
  • Config — 1000 req/min/IP, standard headers, custom error response

5.5 Injection & Data Protection

Intermediate 2 days
  • NoSQL Injection — payload stored as String (not Object), Joi rejects $ operators
  • XSS — escapeHTML() in dashboard, stripUnknown in PATCH
  • PII Redaction — Pino redact paths + redact() / redactPayloadString() utility
  • Payload Size Limit — 1MB cap prevents OOM

6.2 Resilience & Error Handling

Advanced 3 days
  • Error Classification — classifyError() → TRANSIENT vs PERMANENT
  • Exponential Backoff — 1s → 2s → 4s → 8s → 16s retry delays
  • Graceful Shutdown (Ordered Drain):
    1. Stop accepting connections
    2. Drain in-flight HTTP requests
    3. Close database connections
    4. Close Redis
    5. Exit process
  • 15-Second Force-Exit — safety net for hung connections
  • Fail-Fast Boot — crash on missing environment variables
  • uncaughtException / unhandledRejection — catch-all safety nets

6.3 Structured Logging (Pino)

Intermediate 2 days
  • Why Not console.log? — synchronous, blocks event loop at high throughput
  • Pino — async JSON logging via worker thread, 5–10× faster than console
  • Log Levels — fatal, error, warn, info, debug
  • Structured Data — logger.info({ traceId, dbId }, 'message')
  • Built-in Redaction — redact.paths for PII scrubbing at serialization
  • pino-pretty — human-readable output in development
